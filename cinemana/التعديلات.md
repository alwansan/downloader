# التعديلات المضافة على قسم Cinemana فقط

لم يتم لمس `script.js` أو `style.css` الأصليين إطلاقاً — لا حذف ولا تعديل ولا سطر واحد.
كل الميزات أُضيفت بشكل منفصل تماماً حتى لا ينكسر أي شيء موجود.

## الملفات الجديدة (لم تكن موجودة من قبل)
- `cinemana/features.js` → يحتوي كل منطق الميزات الجديدة
- `cinemana/features.css` → تنسيقات الميزات الجديدة فقط

## التعديل الوحيد في ملف موجود
في `cinemana/index.html` تمت إضافة سطرين اثنين فقط (لا شيء غيرهما):
1. سطر لتحميل `features.css` بعد `style.css`
2. سطر لتحميل `features.js` بعد `script.js`

يمكنك التأكد بنفسك عبر أي أداة `diff` أنه لا يوجد أي تغيير آخر في `index.html`.

## الميزات المضافة

### 1) صفحة المواسم والحلقات (شبيهة بصورة 4c)
- خانة بحث برقم حلقة واحد: يظهر فقط الحلقة المطلوبة في كل المواسم التي تحتوي عليها.
- خانة بحث بنطاق (من - إلى): تعرض كل الحلقات الواقعة بين رقمين عبر جميع المواسم.
- كلا البحثين يعملان حتى لو لم يقم المستخدم بفتح (النقر على) الموسم يدوياً أولاً — يتم فتحه تلقائياً عند وجود نتيجة.
- **حفظ تلقائي لكل مسلسل على حِدة**: إذا بحثت عن حلقة معينة أو نطاق معين، ثم دخلت صفحة الجودة (5c) ثم رجعت للخلف، سيبقى البحث كما تركته تماماً دون الحاجة لإعادة كتابته، ودون إعادة إظهار كل الحلقات.

### 2) صفحة اختيار الجودة (شبيهة بصورة 5c)
- شريط أعلى الصفحة يعرض رقم واسم الحلقة الحالية.
- زر "السابق" وزر "التالي" للتنقل بين الحلقات مباشرة دون الرجوع لصفحة المواسم.
- زر "بدء التحميل": يأخذ آخر رابط تم نسخه عبر زر "نسخ الرابط" الأصلي، ويفتحه مباشرة داخل تطبيق **1DM+** باستخدام:
  ```
  intent://<الرابط بدون http/https>#Intent;scheme=https;package=idm.internet.download.manager.plus;end
  ```
  هذه الطريقة (intent://) هي الطريقة القياسية لفتح تطبيق أندرويد معيّن من صفحة ويب داخل المتصفح (تدعمها Chrome وأغلب المتصفحات المبنية على Chromium).

## طريقة عمل الميزات تقنياً (لماذا هي آمنة 100%)
- الاستماع لأحداث النقر يتم بمرحلة "الالتقاط" (capture) فقط لقراءة البيانات، ولا يتم إطلاقاً استخدام `stopPropagation`، لذلك كل أكواد `script.js` الأصلية تستمر بالعمل بالضبط كما هي.
- زرّي "السابق/التالي" لا يعيدان تطبيق أي منطق يدوي؛ بل ينفذّان `click()` حقيقي على عنصر الحلقة نفسه في الصفحة، فيعمل نفس كود الجلب الأصلي بدون أي تغيير.
- رصد الرابط المنسوخ يتم عبر "تغليف" غير مدمّر لدالة `navigator.clipboard.writeText` (تُنفَّذ الدالة الأصلية كما هي دائماً) بالإضافة إلى الاستماع لحدث `copy` القياسي في المتصفح.

## ملاحظة مهمة حول الاختبار
تم اختبار منطق كل ميزة (البحث، النطاق، الحفظ، التنقل، رصد الرابط) عبر محاكاة DOM مطابقة تماماً لأسماء الأصناف (classes) المستخدمة في `style.css` الأصلي، والتأكد أن كل شيء يعمل كما هو متوقع. لم يكن ممكناً تشغيل `script.js` الحقيقي في بيئة الاختبار لأنه يتصل بخوادم خارجية (API) غير متاحة هنا، لذا يُنصح بتجربة الميزات على هاتفك بعد الرفع للتأكد من التوافق الكامل مع البيانات الحقيقية، وإخباري بأي ملاحظة لتعديلها فوراً.
